## Обзор

Apollo — настольный нелинейный медиаредактор. Позволяет импортировать видео, аудио и изображения, раскладывать их по дорожкам на многодорожечном таймлайне и смотреть результат в реальном времени — всё работает нативно через Electron.

## Установка

Требуется **Node.js 18+**.

```bash
# Клонирование репозитория
git clone https://github.com/your-org/apollo.git
cd apollo

# Установка зависимостей
npm install

# Запуск в браузерном режиме (только Vite dev server)
npm run dev

# Запуск как Electron-приложение
npm run start

# Сборка для продакшена
npm run build
```

> `npm run start` запускает Vite и Electron одновременно через `concurrently`. Electron автоматически подключается к `http://localhost:5173`.

## Интерфейс

Интерфейс разделён на две вертикальные зоны:

```
┌─────────────────────────────────────────┐
│  Панель ассетов      │  Окно превью     │
│  (ViewPortAssets)    │  (Preview)       │
├─────────────────────────────────────────┤
│           Таймлайн (TimeLine)           │
│  [Боковая панель] [Треки + Клипы + Плейхед] │
└─────────────────────────────────────────┘
```

| Зона | Компонент | Описание |
|---|---|---|
| Панель ассетов | `ViewPortAssets` | Просмотр, поиск и фильтрация медиафайлов |
| Окно превью | `Preview_Window` | Рендер текущего кадра на позиции плейхеда |
| Таймлайн | `TimeLine` | Многодорожечный редактор клипов |
| Боковая панель | `TrackSidebar` | Имена треков и кнопки удаления |

## Панель ассетов

Панель ассетов — медиабиблиотека текущего проекта.

### Импорт файлов

Перетащите файлы прямо в зону дропа или нажмите кнопку выбора файла (`AddAssets`).  
Принимаемые типы: `image/*`, `video/*`, `audio/*`.

<!-- СКРИНШОТ: Панель с загруженными файлами и строкой поиска -->
> 📸 _Скриншот: заполненная панель ассетов_

### Поиск и фильтрация

После загрузки ассетов появляется строка фильтрации:
- **Строка поиска** — фильтр по имени или ID ассета (debounce 500 мс)
- **Выпадающий список** — показать всё / видео / аудио / изображения

### Переименование ассета

Дважды кликните по имени → редактировать → `Enter` или ✔ для сохранения, `Esc` или ✖ для отмены.

### Перетаскивание на таймлайн

Захватите ассет и бросьте его на дорожку в таймлайне — клип будет создан в позиции дропа.

## Окно превью

Область превью компонует все активные слои на позиции `currentTime` и рендерит их.

- Синхронизация медиа: хук `useMediaSync` отслеживает `currentTime` из `CurrentTimeContext` и позиционирует все медиаэлементы.
- Слоями управляет компонент `Layers` внутри `Preview`.

<!-- СКРИНШОТ: Превью-канвас во время воспроизведения видео -->
> 📸 _Скриншот: окно превью во время воспроизведения_

### Воспроизведение

Переключайте play/pause через панель `Options` или нажимайте `Пробел` при фокусе на таймлайне.

## Таймлайн

Таймлайн — основная рабочая поверхность для монтажа.

<!-- СКРИНШОТ: Таймлайн с несколькими треками и клипами -->
> 📸 _Скриншот: таймлайн с видео-, аудио- и фото-клипами на нескольких дорожках_

### Линейка времени

Кликните в любом месте линейки (`TimeRuler`), чтобы перенести плейхед на эту позицию. Превью обновляется мгновенно.

### Масштаб

Прокручивайте колесо мыши при фокусе на таймлайне для изменения масштаба.  
Хук `useZoomEffect` управляет `scale` (пикселей в секунду) и `STEP` (интервал делений линейки).

### Добавление дорожек

Нажмите **+ add Tracks** в заголовке боковой панели, чтобы добавить новую пустую дорожку.

## Треки

Дорожки — горизонтальные полосы, содержащие клипы. По умолчанию Apollo создаёт **6 треков**.

Каждый трек: `id`, `name`, `clips[]`.

### Цветовая кодировка

| Тип | Цвет | HEX |
|---|---|---|
| Видео | Сине-серый | `#5E7A8C` |
| Аудио | Тёплый коричневый | `#8C6D5E` |
| Изображение | Оливковый | `#718E5E` |
| Текст | Тёмно-коричневый | `#4F3626` |

### Управление дорожками

| Действие | Как |
|---|---|
| Переименовать | Дважды кликнуть по имени → `Enter` сохранить, `Esc` отменить |
| Удалить | Нажать **−** рядом с именем дорожки |
| Добавить | Нажать **+ add Tracks** |

## Клипы

Клип (`ClipItem`) — размещённый на таймлайне медиафрагмент.

**Свойства клипа:** `id`, `assetId`, `start` (секунды), `duration` (секунды), `type`, `sourceOffset`.

<!-- СКРИНШОТ: Выделенный клип с синей рамкой и ручками изменения размера -->
> 📸 _Скриншот: выделенный клип с ручками изменения размера_

### Действия

| Действие | Как |
|---|---|
| Выделить | Клик по блоку клипа (появляется синяя рамка `#4FC3F7`) |
| Переместить | Сначала выделить, затем перетащить на новую позицию (тот же или другой трек) |
| Изменить размер | Перетащить левую или правую ручку для обрезки |
| Переименовать | Иконка редактирования на блоке клипа |
| Затухание аудио | Перетащить ручку затухания у аудиоклипа (`FadeClip`) |

> Перемещение — `useDragClip`, изменение размера — `useResizeClip`.

## Архитектура

Состояние управляется полностью через React Context — без внешних библиотек состояния.

```
main.tsx
└── ClipProvider          (ClipContext)
    └── CurrentTimeProvider  (CurrentTimeContext)
        └── PreviewProvider  (PreviewContext)
            └── App
                ├── VIewPort
                │   ├── ViewPortAssets
                │   └── Preview_Window → Preview
                └── TimeLine
                    ├── Options
                    ├── TrackSidebar[]
                    ├── TimeRuler
                    ├── Tracks → TrackRow → ClipItem[]
                    └── Playhead
```

### Контексты

| Контекст | Зона ответственности |
|---|---|
| `ClipContext` | Ассеты, клипы таймлайна, треки, ID выделенного клипа |
| `CurrentTimeContext` | Позиция плейхеда (`currentTime` в секундах) |
| `PreviewContext` | Состояние воспроизведения (`isPlay`), `handlePlay`, `handlePause` |

### Electron

`electron/main.ts` создаёт `BrowserWindow` (1200 × 800) и в режиме разработки загружает `http://localhost:5173`.
